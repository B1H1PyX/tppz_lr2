# Інформаційна система оцінювання стану динамічного об'єкту

Цей репозиторій містить код лендінг-сторінки для бакалаврської роботи. Сайт є повністю статичним (HTML, CSS, JavaScript).

## Початок роботи (для розробників)

Ця інструкція допоможе вам швидко розгорнути проект локально для розробки.

### Передумови

Вам знадобиться лише два інструменти:
1.  **Git** - для клонування репозиторію.
2.  **Сучасний веб-браузер** (Chrome, Firefox, Edge, Safari) - для перегляду сайту.

Бажано, але не обов'язково, мати кодовий редактор (VS Code, Sublime Text тощо).

### Покрокова інструкція

1.  **Встановіть Git (якщо ще не встановлено):**
    *   Перейдіть на офіційний сайт: [https://git-scm.com/downloads](https://git-scm.com/downloads)
    *   Завантажте версію для вашої ОС та встановіть, залишаючи всі налаштування за замовчуванням.

2.  **Клонуйте репозиторій:**
    Відкрийте термінал (командний рядок) та виконайте команду:
    ```bash
    git clone https://github.com/B1H1PyX/tppz_lr2.git
    cd tppz_lr2
    ```
    *(За потреби замініть URL на актуальний)*

3.  **Відкрийте проект:**
    *   Ви можете відкрити папку `tppz_lr2` у вашому текстовому редакторі.
    *   Щоб переглянути сайт, просто двічі клацніть на файл `index.html`. Він відкриється у вашому браузері за замовчуванням.

4.  **Налаштування середовища розробки (опціонально, для зручності):**
    *   Якщо ви використовуєте VS Code, рекомендовано встановити розширення `Live Server` (автор: Ritwick Dey).
    *   Після встановлення, натисніть `Go Live` у правому нижньому кутку VS Code. Це запустить локальний веб-сервер, і сторінка буде доступна за адресою `http://127.0.0.1:5500`. Головна перевага — автоматичне оновлення сторінки при зміні файлів.

### Робота з проектом

*   **Структура папок:**
    *   `/` - корінь проекту (містить `index.html`)
    *   `/css/` - файли стилів (`style.css`, `normalize.css`)
    *   `/scripts/` - JavaScript файли (`script.js`)
    *   `/img/` - зображення для сайту
    *   `/docs/` - технічна документація (для DevOps)

*   **Редагування:**
    *   Змінюйте HTML у `index.html`.
    *   Стилі знаходяться у `css/style.css`.
    *   JavaScript логіка — у `scripts/script.js`.

*   **Формування коммітів:**
    Після внесення змін, використовуйте стандартні команди Git:
    ```bash
    git add .
    git commit -m "Опис ваших змін"
    git push
    ```

### Налагодження

*   Для перевірки коду та помилок використовуйте інструменти розробника у вашому браузері (F12).
    *   Вкладка "Console" показує помилки JavaScript.
    *   Вкладка "Network" показує, які файли завантажуються.
    *   Вкладка "Elements" дозволяє переглядати та змінювати HTML/CSS в реальному часі.
